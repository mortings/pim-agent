MCP Roadmap

Bluestone as a toolbox

Making the MCP a complete, safe, governed capability surface — so any UI or agent can operate Bluestone PIM without ever touching the backend.

North star
Reads are largely done.
The value — and the moat — is writes done safely.
● Rich reads & completeness shipped ● Write surface thin ● Governance still UI-side

Proven this session: ~160 lines flipped a full, vibe-coded PIM UI onto the live tenant via MCP. The front-of-house is cheap. The rest of this deck is how the toolbox earns production trust.

1
Phase 1

Close the write surface

turn “browse” into “operate”
What ships
  • {C} Per-market attribute writes (incl. clear / unset)
  • {C} Lifecycle & publish — submit, approve, publish, archive, publish-to-channel
  • {C} Relationships & variants (cross/up-sell, bundles, variant matrix)
  • {C} Assets — upload, link, alt-text (DAM write)
  • {C} Guarded delete / archive
Why it matters

Today you can read the tenant but barely change it. This is what makes it an actual toolbox rather than a viewer.

2
Phase 2 · ship alongside Phase 1

Governance & safety

safe in untrusted & agentic hands
What ships
  • {C} Per-token RBAC / ABAC enforced server-side
  • {C} Validation + dry-run / preview on writes
  • {C} Idempotency keys
  • {C} Rate limits & per-tenant isolation
  • {C} Audit & attribution on every change
Why it matters

The moment untrusted UIs and non-deterministic agents can write, governance can’t live in the UI. This is the gate that lets you open the toolbox at all.

3
Phase 3

Scale & enterprise depth

1M+ products · 150+ languages
What ships
  • {C} Bulk / batch + async jobs (import, mass-edit, re-publish)
  • {C} Cursor pagination everywhere
  • {C} Completeness & validation rules
  • {C} Localization / TMS workflow tools
Why it matters

Survives the real catalog. Per-item tool calls won’t scale to a million products across 150 languages.

4
Phase 4

Agent-native ecosystem

toolbox becomes a platform
What ships
  • {C} Event stream / webhooks (react, don’t poll)
  • {C} Stable, versioned, discoverable schemas + capability manifest
  • {C} Scoped app / agent tokens for 3rd parties
  • {C} Change-set / transaction + confirm-before-write semantics
Why it matters

Humans, partners and agents build on Bluestone without breaking as it evolves. The moat shifts from “our UI” to “the safest platform to do PIM.”

The decision for this meeting
Ship governance in lockstep with the write surface.

Phases 1 + 2 together. Writes without per-token governance is a liability. Writes with it is the differentiated, agent-ready platform the narrative promises.

Bluestone as a toolbox

Same backend. Infinite fit-for-purpose UIs and agents on top.
The work that wins is making the MCP a complete, safe, governed write surface.
